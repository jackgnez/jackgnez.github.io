如何用hexo+github搭建自己的博客

github 允许我们在用户名.github.io这种格式的域名下访问一个自己的静态页面，但如果我们只是自己写一个静态页面就想要来当博客网站的话，每次更新会非常麻烦。所以有人写了一种，在本地写文章，然后自动更新整个网站代码的程序，供我们在 github 上搭建自己的博客网站。其中一个是 jekyll ，另一个就是我们要介绍的 hexo 。本文主要介绍如何在 github 上搭建自己的博客以及如何舒服地写博客。

1 总体流程

总的来说，我们需要把博客的代码放到github的命名为用户名.github.io仓库里，然后通用户名.github.io就可以访问这个博客啦。hexo已经帮我们把代码都写好了，也就是说我们自己不用生产代码，我们只是代码的搬运工。每次更新文章我们只需要先把文章写好再调用hexo的命令去更新整个博客的代码，再将代码push到github上面就大功告成了。
所以，总的来说，整体的流程是这样的：

4 部署 hexo 代码到 github 上

要部署 hexo 代码到 github 上，我们就需要和 github 取得连接，连接的常用方式有两种，一种是 ssh 连接，另一种是直接输入账号密码连接，如果不设置 ssh 连接，应该就是输入账号密码连接了（我记得我没设置的时候，就是用账号密码连接的）。我现在设置了 ssh ，就讲一讲用 ssh 连接的方法吧。ssh 连接其实意思就是，有一把钥匙，只有你本地才有，把这把钥匙给了github，这样github就可以确定，这个代码肯定是由你（这台电脑）上传的，而不是别人。

5 写博客和更新博客

hexo 需要 markdown 的 md 格式的文件写博客，它会把 md 格式的文章转换成 html 页面，具体的细节我也不知道，不过我想也不用管。大家可以随意使用网上各种 markdown 编辑器。
但这里有一个问题，我们都知道在网上写博客，图片是需要引用图片地址或者先本地上传到博客网站再引用的，但是大多数写博客的markdown 编辑器都不支持上传将图片上传到某个网站上，而免费上传图片的大部分保留时间是有限的。为了解决这个问题，我们需要通过以下修改来完成上传图片的功能：
